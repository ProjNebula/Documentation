# Match States `<states>`

This module can be used to modify states that the match goes through.

## State `<state>`

A state that will be reached in the lifetime of the match.

```xml

<states>
  <state id="play" playing="true" />
</states>

```

##### State Attributes

| Attribute | Description | Type | Default |
| --- | --- | --- | --- |
| `id` | The globally unique ID of this feature. This can be used to reference this feature from other parts of the XML. | <span class="badge badge--primary">Feature ID</span> | |
| `playing` | If the match should be marked as playing during this state. | <span class="badge badge--primary">True/False</span> | `true` |
